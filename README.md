# Second Brain

Obsidian-facing workspace for ChipBoss/OpenClaw memory, notes, project collateral, and research.

## Where Content Goes

- `Inbox/` - quick captures, rough notes, and unsorted ideas.
- `Daily/` - date-based logs and session notes.
- `Projects/` - formal project material, organised under `Projects/projects/<project-slug>/`.
- `Research/` - exploratory research, marketing requests, campaign ideas, audience/channel research, and lightweight market validation.
- `Decisions/` - durable decisions that affect projects, architecture, workflow, or operating rules.
- `People/` - stable notes about people, preferences, and working context.
- `Resources/` - reference material, provider notes, model/config notes, and reusable external context.
- `Templates/` - reusable note templates.
- `Archive/` - old or superseded notes that should remain available but out of the active flow.
- `00-agent/` - agent-facing operating rules and retrieval guidance.

## Project Notes

Store project collateral inside `Projects/projects/<project-slug>/`. Keep buildable source code outside the vault, usually under `/source-code/<project-slug>/`, and link to it from the project note when useful.

Use project folders for briefs, specs, implementation plans, roadmaps, decisions, design notes, release notes, and project-specific research.

Use `Research/` instead of `Projects/` when the request is exploratory, marketing-led, or not yet tied to a formal project deliverable.
