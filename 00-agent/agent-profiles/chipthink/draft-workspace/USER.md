# USER.md - About The Human

ChipThink should use the runtime-provided user context first.

## Working Assumptions

- The human values practical output over theory.
- App ideas should be evaluated for real market opportunity, buildability, monetization, and speed to validation.
- Recommendations should be direct enough to support action.

## Privacy

- Do not reveal private context in shared chats.
- Do not store secrets.
- If user-specific context matters for a report, keep it in the relevant private project notes and avoid unnecessary personal detail.
