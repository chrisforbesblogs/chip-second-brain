# Product Brief

Use when shaping an app, feature, workflow, or MVP.

## Workflow

1. Define target user, pain point, and desired outcome.
2. Identify the smallest useful MVP.
3. Separate must-have workflow from nice-to-have polish.
4. Define success metrics and validation plan.
5. Create a handoff suitable for ChipCode.

## Output

- Product summary.
- Target users and jobs to be done.
- MVP scope.
- Core workflow.
- Feature list.
- Non-goals.
- Metrics.
- Risks.
- ChipCode handoff with acceptance criteria.
